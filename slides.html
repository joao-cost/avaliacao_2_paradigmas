Conceituação

Paradigma Imperativo / Procedural

Características Principais

  • Foco na Execução: O programador prescreve uma sequência exata de passos e comandos (como uma receita).
  • Separação de Elementos: Os dados (estruturas) ficam separados dos procedimentos (funções) que os operam.
  • Controle de Fluxo: Guiado fortemente por laços (`for`, `while`) e desvios condicionais (`if`, `switch`).

No modelo procedural, as funções agem de fora sobre as estruturas de dados que lhes são enviadas.

Este paradigma é amplamente encontrado em ferramentas de automação, scripts rápidos de sistema e na tradicional linguagem C clássica.

Conceituação

Paradigma Orientado a Objetos

A Programação Orientada a Objetos (POO) surge com a ideia de aproximar o código de elementos e conceitos do mundo real.

Em vez de organizar o sistema como uma esteira de funções que operam dados soltos, modelamos entidades autônomas chamadas objetos.

Pilares da POO

  • Encapsulamento: Proteção e união de dados (atributos) e código (métodos) na mesma estrutura.
  • Herança: Especialização de classes gerais para filhas (reuso de atributos/comportamentos).
  • Polimorfismo: Permite que um mesmo comportamento se comporte de formas distintas em classes filhas.

Demonstração Prática

Diferença de Código: Procedural vs Orientado a Objetos

Estruturado (C) 
typedef struct {
    char nome[50];
    int idade;
} Animal;

void imprimir(Animal a) {
    printf("Nome: %s\n", a.nome);
}

int main() {
    Animal a = {"Simba", 7};
    imprimir(a); // Dados enviados à função
    return 0;
}
Orientado a Objetos (Java) 
public class Animal {
    String nome;
    int idade;

    Animal(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }

    void imprimir() {
        System.out.println(nome);
    }
}

// Uso no código principal:
Animal a = new Animal("Simba", 7);
a.imprimir(); // O próprio objeto age

Estudo de Caso

Como isso se aplica no nosso Sistema?

O nosso projeto do Zoológico em Java utiliza uma arquitetura híbrida de fato, embora a modelagem de dados seja puramente estruturada em POO.

  • Classes de Domínio (POO): `Animal`, `Tratador` e as subclasses (`Leao`, `Cobra` etc.) representam objetos reais, aplicando Herança (`extends`) e Polimorfismo.
  • Fluxo de Menu (Procedural): A classe `Main.java` contém funções e laços tradicionais que decidem o fluxo imperativo de telas e ações.

├── Leao.java / Cobra.java (Heranças)
├── Zoologico.java (Gerenciamento OO)
└── Main.java (Fluxo Iterativo Imperativo)

Arquivos do Projeto
class Animal {
    int id;
    String nome;
    String especie;
    int idade;

    Animal(int id, String nome, String especie, int idade) {
        this.id = id;
        this.nome = nome;
        this.especie = especie;
        this.idade = idade;
    }

    void imprimir() {
        System.out.println("ID: " + id);
        System.out.println("Nome: " + nome);
        System.out.println("Espécie: " + especie);
    }
}
// Demonstração Prática de Herança em Java
class Leao extends Animal {

    Leao(int id, String nome, int idade) {
        super(id, nome, "Leão", idade); // Reuso do Construtor Pai
    }

    void rugir() {
        System.out.println(nome + " está rugindo");
    }

    @Override
    void imprimir() {
        System.out.println("=== Leão ===");
        super.imprimir(); // Polimorfismo e extensão
        rugir();
    }
}
import java.util.ArrayList;

class Zoologico {
    ArrayList<Animal> animais = new ArrayList<>();

    void cadastrarAnimal(Animal a) {
        animais.add(a); // Armazena referências de Objetos
    }

    Animal buscarAnimalPorId(int id) {
        for (Animal a : animais) {
            if (a.id == id) return a;
        }
        return null;
    }
}
public class Main {
    static Zoologico zoo = new Zoologico();

    public static void main(String[] args) {
        int opcao;
        do {
            mostrarMenu();
            opcao = lerOpcao();
            
            // Fluxo puramente imperativo/procedural
            switch (opcao) {
                case 1: cadastrarAnimal(); break;
                case 2: zoo.listarAnimais(); break;
                case 0: System.out.println("Saindo..."); break;
            }
        } while (opcao != 0);
    }
}
